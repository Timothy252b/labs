Что такое поток? Поток это абстракция последовательности байтов, используемая для ввода или вывода данных, которая связывает программу с источником или приемником данных например файлом или консолью.

Какие типы потоков существуют? Существуют стандартные потоки ввода вывода ошибок, файловые потоки для работы с файлами, строковые потоки для работы со строками в памяти.

Какую библиотеку надо подключить при использовании стандартных потоков? Библиотеку iostream.

Какую библиотеку надо подключить при использовании файловых потоков? Библиотеку fstream.

Какую библиотеку надо подключить при использовании строковых потоков? Библиотеку sstream.

Какая операция используется при выводе в форматированный поток? Оператор вставки.

Какая операция используется при вводе из форматированных потоков? Оператор извлечения.

Какие методы используются при выводе в форматированный поток? Методы put для вывода одного символа, write для вывода блока данных.

Какие методы используются при вводе из форматированного потока? Методы get для чтения одного символа, getline для чтения строки, read для чтения блока данных.

Какие режимы для открытия файловых потоков существуют? Режимы ios in для чтения, ios out для записи, ios app для добавления, ios ate для перехода в конец при открытии, ios trunc для усечения файла, ios binary для бинарного режима.

Какой режим используется для добавления записей в файл? Режим ios app.

Какой режим используется в конструкторе ifstream file f txt? Режимы ios in по умолчанию.

Какой режим используется в конструкторе fstream file f txt? Режим ios in Out по умолчанию.

Какой режим используется в конструкторе ofstream file f txt? Режим ios out по умолчанию.

Каким образом открывается поток в режиме ios out ios app? Открывается только для записи, данные добавляются в конец файла, старая информация не удаляется.

Каким образом открывается поток в режиме ios out ios trunc? Открывается только для записи, файл усекается до нулевой длины или создается новый.

Каким образом открывается поток в режиме ios out ios in ios trunc? Открывается для чтения и записи, файл усекается до нулевой длины.

Каким образом можно открыть файл для чтения? Созданием объекта ifstream с именем файла, или открытием fstream с режимом ios in.

Каким образом можно открыть файл для записи? Созданием объекта ofstream с именем файла, или открытием fstream с режимом ios out.

Привести примеры открытия файловых потоков в различных режимах. ifstream fin data txt ios in, ofstream fout report txt ios out ios trunc, fstream fio data dat ios in ios out ios binary, ofstream log txt ios app.

Привести примеры чтения объектов из потока. fin переменная, fin read reinterpret cast char объект sizeof объект, getline fin строка.

Привести примеры записи объектов в поток. fout переменная, fout write reinterpret cast char объект sizeof объект, fout put символ.

Сформулировать алгоритм удаления записей из файла. Открыть исходный файл для чтения, создать временный файл для записи, читать записи из исходного файла, если запись не подлежит удалению записывать ее во временный файл, после обработки всех записей закрыть оба файла, удалить исходный файл, переименовать временный файл в имя исходного.

Сформулировать алгоритм добавления записей в файл. Открыть файл в режиме ios app или ios out ios app, установить позицию записи в конец файла, записать новые данные. Для добавления в середину прочитать весь файл в память, вставить записи в нужном месте, перезаписать файл.

Сформулировать алгоритм изменения записей в файле. Если записи имеют фиксированный размер открыть файл в режиме ios in ios out, найти позицию нужной записи, переместить указатель на эту позицию, записать измененные данные. Если размер переменный прочитать все записи в память, изменить нужные, перезаписать весь файл.
