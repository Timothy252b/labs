Для чего используется механизм наследования? Механизм наследования используется для создания новых классов на основе существующих, что позволяет повторно использовать код, добавлять новую функциональность и организовывать иерархии классов с отношением является.

Каким образом наследуются компоненты класса, описанные со спецификатором public? При открытом наследовании public-члены базового класса становятся public-членами производного класса, protected-члены базового класса становятся protected-членами производного класса, private-члены базового класса не наследуются и остаются недоступными.

Каким образом наследуются компоненты класса, описанные со спецификатором private? При закрытом наследовании public и protected-члены базового класса становятся private-членами производного класса, private-члены базового класса не наследуются.

Каким образом наследуются компоненты класса, описанные со спецификатором protected? При защищенном наследовании public и protected-члены базового класса становятся protected-членами производного класса, private-члены базового класса не наследуются.

Каким образом описывается производный класс? Производный класс описывается с указанием базового класса после двоеточия и спецификатора доступа class Derived public Base.

Наследуются ли конструкторы? Конструкторы не наследуются, кроме конструкторов наследования в C++11.

Наследуются ли деструкторы? Деструкторы не наследуются.

В каком порядке конструируются объекты производных классов? Сначала вызывается конструктор базового класса, затем конструктор производного класса.

В каком порядке уничтожаются объекты производных классов? Сначала вызывается деструктор производного класса, затем деструктор базового класса.

Что представляют собой виртуальные функции и механизм позднего связывания? Виртуальные функции это функции-члены класса, которые могут быть переопределены в производных классах. Механизм позднего связывания означает, что вызов виртуальной функции разрешается во время выполнения на основе фактического типа объекта, а не типа указателя или ссылки.

Могут ли быть виртуальными конструкторы? Деструкторы? Конструкторы не могут быть виртуальными. Деструкторы могут и должны быть виртуальными в полиморфных базовых классах.

Наследуется ли спецификатор virtual? Спецификатор virtual наследуется, если функция объявлена виртуальной в базовом классе, она остается виртуальной во всех производных классах.

Какое отношение устанавливает между классами открытое наследование? Открытое наследование устанавливает отношение является, то есть объект производного класса является разновидностью объекта базового класса.

Какое отношение устанавливает между классами закрытое наследование? Закрытое наследование устанавливает отношение реализован посредством, то есть производный класс использует реализацию базового класса, но не является его подтипом.

В чем заключается принцип подстановки? Принцип подстановки гласит, что везде, где ожидается объект базового класса, можно использовать объект производного класса, и программа будет работать корректно.

Имеется иерархия классов Student с полями int age и public string name, Employee наследует Student открыто и добавляет protected string post, Teacher наследует Employee и добавляет protected int stage. Объект x типа Teacher будет иметь компонентные данные: name public из Student, age private из Student через механизмы наследования недоступен напрямую но присутствует, post protected из Employee, stage protected из Teacher.
