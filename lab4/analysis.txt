Необходимо разработать иерархию классов для представления человека и студента. Базовый класс Person должен содержать два защищенных поля: имя в виде строки и возраст в виде целого числа. Защищенный уровень доступа выбран для того, чтобы производные классы имели прямой доступ к этим полям.

В классе Person требуется реализовать три конструктора. Конструктор без параметров инициализирует имя значением по умолчанию, например Jane Doe, а возрастом значением -1, что может сигнализировать о неопределенном состоянии. Конструктор с параметрами принимает имя и возраст и инициализирует соответствующие поля. Конструктор копирования создает новый объект на основе существующего. Деструктор должен быть виртуальным, чтобы обеспечить корректное разрушение объектов производных классов через указатель на базовый класс.

Для доступа к полям необходимы методы setName, getName, getAge. Метод setName изменяет имя, getName возвращает имя, getAge возвращает возраст. Методы getName и getAge не должны изменять объект, поэтому их следует объявить как константные.

Необходимо перегрузить оператор присваивания для копирования данных из одного объекта Person в другой. Также требуется перегрузить потоковые операторы ввода и вывода. Оператор вывода должен выводить имя и возраст в удобочитаемом формате. Оператор ввода запрашивает у пользователя ввод имени и возраста.

В классе Person следует перегрузить операторы инкремента. Префиксный инкремент увеличивает возраст на единицу и возвращает ссылку на текущий объект. Постфиксный инкремент создает копию объекта до изменения, увеличивает возраст и возвращает созданную копию.

Метод print должен выводить на экран имя и возраст. Этот метод должен быть виртуальным, чтобы производные классы могли его переопределить.

Производный класс Student наследует от Person. Он добавляет два защищенных поля: предмет в виде строки и оценку в виде целого числа. Конструктор без параметров вызывает конструктор базового класса по умолчанию и инициализирует предмет значением Astrology, а оценку значением -1. Конструктор с параметрами принимает имя, возраст, предмет и оценку, передает имя и возраст в конструктор базового класса и инициализирует собственные поля. Конструктор копирования вызывает конструктор копирования базового класса и копирует предмет и оценку. Деструктор не требует специальной реализации.

Оператор присваивания Student должен копировать поля базового класса и собственные поля. Методы setSubject и setMark изменяют соответствующие поля. Метод getSubject и getMark для доступа к полям также желательно реализовать.

Потоковые операторы ввода и вывода для Student должны выводить информацию о студенте, включая данные из базового класса. Оператор вывода может использовать оператор вывода базового класса для вывода имени и возраста, затем добавлять информацию о предмете и оценке. Оператор ввода должен запрашивать ввод данных базовой части через оператор ввода Person, а затем запрашивать предмет и оценку.

Метод print должен быть переопределен: сначала вызывать метод print базового класса, затем выводить предмет и оценку. Метод aboutMark анализирует значение оценки и выводит сообщение о неудовлетворительной оценке, если оценка меньше трех.

Демонстрационная программа должна создавать объекты Student с помощью всех трех конструкторов. Также создать объект Person. Показать принцип подстановки: указатель на базовый класс может указывать на объект производного класса. Для этого следует объявить указатель типа Person* и присвоить ему адрес объекта Student. При выводе через этот указатель благодаря виртуальности метода print будет вызвана версия метода из класса Student.

Программа должна демонстрировать работу перегруженных операторов ввода и вывода для Person и Student, операторов инкремента для Person, операторов присваивания, а также метода aboutMark для проверки оценки. Получение и возврат объекта базового класса может быть продемонстрирован через функции, принимающие ссылку на базовый класс, что также показывает принцип подстановки.
