В чем смысл использования шаблонов? Шаблоны позволяют создавать обобщенные функции и классы, работающие с различными типами данных без дублирования кода, что обеспечивает повторное использование и типобезопасность.

Каковы синтаксис и семантика шаблонов функций? Шаблон функции начинается с ключевого слова template, затем в угловых скобках список параметров шаблона с ключевым словом class или typename, далее объявление функции. При вызове компилятор автоматически выводит типы аргументов и инстанцирует нужную версию функции.

Каковы синтаксис и семантика шаблонов классов? Шаблон класса начинается с ключевого слова template, затем в угловых скобках список параметров шаблона, далее объявление класса. При создании объекта необходимо явно указывать типы параметров в угловых скобках. Компилятор генерирует конкретный класс для каждого набора типов.

Что такое параметры шаблона функции? Параметры шаблона функции это формальные имена типов или значений, которые будут заменены конкретными типами или значениями при инстанцировании шаблона.

Перечислите основные свойства параметров шаблона функции. Параметр шаблона может быть типом typename или class, может быть константным целочисленным значением, может иметь значение по умолчанию. Разные параметры могут быть разных видов. При инстанцировании типы выводятся из аргументов вызова функции.

Как записывать параметр шаблона? Параметр шаблона записывается внутри угловых скобок после ключевого слова template, например template typename T или template int N или template class T.

Можно ли перегружать параметризованные функции? Да, можно перегружать как обычные функции с разными параметрами, так и сами шаблоны функций, а также сочетать шаблонные и обычные перегруженные версии.

Перечислите основные свойства параметризованных классов. Параметризованный класс является обобщенным. Разные инстанцирования с разными типами это разные типы. Определения методов можно помещать как внутри так и вне класса. Можно использовать любые типы, поддерживающие операции, используемые в методах класса. Шаблон можно специализировать для конкретных типов.

Все ли компонентные функции параметризованного класса являются параметризованными? Да, все компонентные функции параметризованного класса являются параметризованными, потому что они зависят от параметров шаблона класса, даже если сами не имеют отдельных параметров шаблона.

Являются ли дружественные функции, описанные в параметризованном классе, параметризованными? Дружественная функция, объявленная внутри параметризованного класса, не обязательно является параметризованной. Если она объявлена без указания параметров шаблона, то для каждого инстанцирования класса создается отдельная обычная функция.

Могут ли шаблоны классов содержать виртуальные компонентные функции? Да, шаблоны классов могут содержать виртуальные компонентные функции, но сами виртуальные функции не могут быть шаблонными. Каждое инстанцирование класса порождает набор виртуальных функций.

Как определяются компонентные функции параметризованных классов вне определения шаблона класса? При определении функции вне класса необходимо повторить объявление template с теми же параметрами, затем указать квалифицированное имя с параметрами шаблона в угловых скобках, например template class T void MyClass T myFunction.

Что такое инстанцирование шаблона? Инстанцирование шаблона это процесс генерации компилятором реального кода функции или класса на основе шаблона и конкретных переданных типов или значений параметров.

На каком этапе происходит генерирование определения класса по шаблону? Генерирование определения класса по шаблону происходит на этапе компиляции в момент первого использования шаблона с конкретными параметрами.
