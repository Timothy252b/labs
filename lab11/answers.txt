Из каких частей состоит библиотека STL? Библиотека STL состоит из контейнеров для хранения данных, итераторов для доступа к элементам, алгоритмов для обработки данных, функциональных объектов и адаптеров контейнеров.

Какие типы контейнеров существуют в STL? В STL существуют последовательные контейнеры vector, list, deque, array, forward_list, ассоциативные контейнеры set, multiset, map, multimap, неупорядоченные ассоциативные контейнеры unordered_set, unordered_multiset, unordered_map, unordered_multimap, а также адаптеры контейнеров stack, queue, priority_queue.

Что нужно сделать для использования контейнера STL в своей программе? Подключить соответствующий заголовочный файл, например vector, list, и добавить using namespace std или явно использовать префикс std.

Что представляет собой итератор? Итератор это объект, который позволяет обходить элементы контейнера последовательно, обеспечивая единообразный доступ к элементам разных типов контейнеров.

Какие операции можно выполнять над итераторами? Над итераторами можно выполнять операции инкремента для перехода к следующему элементу, декремента для перехода к предыдущему, разыменования для доступа к значению, сравнения на равенство и неравенство, а также сложение и вычитание с целыми числами для итераторов произвольного доступа.

Каким образом можно организовать цикл для перебора контейнера с использованием итератора? Использовать цикл for с итератором от container.begin до container.end, либо применить цикл for с автоматическим выводом типа for auto it container begin it container end it.

Какие типы итераторов существуют? Существуют итераторы ввода, итераторы вывода, однонаправленные итераторы, двунаправленные итераторы, итераторы произвольного доступа.

Какие операции являются эффективными для контейнера vector Почему? Для vector эффективны операции доступа по индексу за время O 1, добавление и удаление в конец с амортизированным временем O 1, потому что vector хранит элементы в непрерывной области памяти.

Какие операции являются эффективными для контейнера list Почему? Для list эффективны операции вставки и удаления в любой позиции за время O 1, потому что list реализован как двусвязный список и не требует сдвига элементов.

Какие операции являются эффективными для контейнера deque Почему? Для deque эффективны доступ по индексу за время O 1, добавление и удаление в начале и конце за амортизированное время O 1, потому что deque реализован как последовательность блоков памяти.

Перечислить методы, которые поддерживает последовательный контейнер list. Методы list включают assign, back, begin, clear, empty, end, erase, front, insert, merge, pop_back, pop_front, push_back, push_front, remove, remove_if, reverse, size, sort, splice, swap, unique.

Что представляют собой адаптеры контейнеров? Адаптеры контейнеров это классы, которые предоставляют ограниченный интерфейс к базовому контейнеру, преобразуя его в структуру с определенной семантикой доступа, например stack, queue, priority_queue.

Чем отличаются друг от друга объявления stack int s и stack int list int s? stack int s использует стандартный контейнер deque, а stack int list int s использует list в качестве базового контейнера.

Чем отличаются друг от друга контейнеры queue и priority_queue? queue обслуживает элементы в порядке FIFO первый вошел первый вышел, а priority_queue извлекает элемент с наивысшим приоритетом согласно заданному критерию сравнения.

Задан контейнер stack. Как удалить из него элемент с заданным номером? В стеке нельзя удалить элемент с заданным номером напрямую. Нужно извлекать элементы из стека до нужного, сохраняя их в другой стек, затем пропустить удаляемый, и вернуть сохраненные элементы обратно.

Задан контейнер queue. Как удалить из него элемент с заданным номером? В очереди нельзя удалить элемент с заданным номером напрямую. Нужно извлекать элементы из очереди до нужного и помещать их обратно в конец, затем удалить нужный элемент.
