Требуется разработать иерархию классов для представления печатных изданий. Абстрактный базовый класс PRINT должен содержать защищенные поля: название издания в виде строки и автора в виде строки. Конструктор по умолчанию инициализирует обе строки пустыми значениями. Конструктор с параметрами принимает название и автора. Деструктор должен быть виртуальным для обеспечения корректного разрушения объектов производных классов. В классе определяются два чистых виртуальных метода: input для ввода данных и output для вывода информации об объекте. Также определяются виртуальные методы getName и getAuthor для доступа к полям.

Производный класс MAGAZIN наследует от PRINT и добавляет закрытое поле pages для хранения количества страниц. Конструктор без параметров вызывает конструктор базового класса и инициализирует страницы нулем. Конструктор с параметрами принимает название, автора и количество страниц. Метод input переопределяется и последовательно запрашивает ввод названия, автора и количества страниц. Метод output переопределяется и выводит информацию о журнале в квадратных скобках. Метод getName возвращает название.

Для организации группы объектов на основе структуры дерева определяется структура TreeNode. Каждый узел содержит указатель на объект MAGAZIN, вектор указателей на дочерние узлы children и указатель на родительский узел parent. Конструктор инициализирует данные и обнуляет родителя. Деструктор удаляет данные журнала и рекурсивно всех потомков.

Класс Tree является основным классом группы. Он содержит защищенные поля: root указатель на корневой узел и size целочисленное количество элементов. Конструктор инициализирует корень нулем и размер нулем. Деструктор вызывает метод clear для рекурсивного удаления всех узлов. Метод addRoot создает корневой узел из переданного журнала, если корень еще не существует. Метод addChild находит узел с заданным именем и добавляет к нему новый дочерний узел. Метод removeNode удаляет узел по имени с учетом того, что корень удаляется отдельно. Метод findNode выполняет рекурсивный поиск узла по названию. Методы printTree и printNames рекурсивно выводят информацию о структуре и о названиях журналов. Метод display вызывает printTree. Метод displayNames вызывает printNames. Метод clear рекурсивно удаляет все узлы. Оператор тильда возвращает размер дерева.

Класс Dialog наследует от Tree и реализует интерактивное меню для работы с деревом. Метод run содержит цикл обработки команд. Команда m создает корневой журнал, если корень отсутствует. Команда плюс добавляет дочерний журнал к указанному родительскому узлу. Команда минус удаляет журнал по указанному имени. Команда s выводит полную структуру дерева с отступами. Команда z выводит только названия всех журналов. Команда q завершает программу.

В основной программе создается объект класса Dialog и вызывается его метод run. Пользователь последовательно вводит команды для создания дерева, добавления и удаления узлов, просмотра структуры. Программа поддерживает вывод текущего размера группы после каждой операции. Все методы ввода и вывода объектов журнала используют переопределенные виртуальные функции, что обеспечивает полиморфное поведение. При необходимости в будущем могут быть добавлены другие производные классы от PRINT, которые будут корректно обрабатываться существующей структурой дерева.
