Класс-группа — это класс, предназначенный для хранения и управления набором однородных или разнородных объектов. Примеры: список, стек, очередь, массив, множество, словарь.

Пример описания класса List:
class List {
private:
    struct Node {
        int data;
        Node* next;
    };
    Node* head;
    int size;
public:
    List();
    List(int value);
    List(const List& other);
    ~List();
    void display();
};

Конструктор без параметров: List::List() : head(nullptr), size(0) {}
Конструктор с параметром: List::List(int value) : head(nullptr), size(0) { head = new Node{value, nullptr}; size = 1; }
Конструктор копирования: List::List(const List& other) : head(nullptr), size(0) { Node* current = other.head; while (current) { add(current->data); current = current->next; } }

Деструктор: List::~List() { while (head) { Node* temp = head; head = head->next; delete temp; } size = 0; }

Метод для просмотра элементов: void List::display() { Node* current = head; while (current) { cout << current->data << " "; current = current->next; } cout << endl; }

Группа дает иерархию типа «часть-целое» (агрегацию или композицию), а не наследование. Это иерархия контейнеров, где один объект содержит другие.

Абстрактный класс задает общий интерфейс для всех объектов, которые могут храниться в группе. Это позволяет группе работать с разнородными объектами единообразно через указатели или ссылки на базовый класс, не зная их конкретных типов.

Событие — это факт возникновения некоторого действия. События используются для уведомления одних компонентов программы о действиях других компонентов или пользователя, обеспечивая реактивность и асинхронное взаимодействие.

Характеристики события-сообщения: тип события, источник события, время возникновения, данные (параметры события) и, возможно, адресат.

Пример структуры, описывающей событие: struct Event { int type; int x; int y; int keyCode; };

Полю what присваиваются числовые коды, идентифицирующие тип события. Например: evMouseMove — движение мыши, evMouseDown — нажатие кнопки мыши, evKeyDown — нажатие клавиши, evMessage — пользовательское сообщение.

Полю command присваиваются значения (коды команд), когда what указывает на событие типа команда. Например: cmOpen — открыть файл, cmSave — сохранить, cmExit — выход из программы.

Поля a и message используются для передачи дополнительных параметров команды. message может содержать текст сообщения или дополнительный код подкоманды, а a — числовой параметр.

Необходимы методы: GetEvent() для получения события, DispatchEvent() или HandleEvent() для маршрутизации и обработки события, ClearEvent() для очистки или сброса события, а также метод проверки наличия событий.

Главный цикл обработки событий:
while (running) {
    if (GetEvent(&event)) {
        HandleEvent(&event);
        if (event.type == EV_QUIT) running = false;
    } else {
        // idle actions
    }
    ClearEvent(&event);
}

ClearEvent() очищает или сбрасывает событие после его обработки. Обычно он обнуляет поля структуры события или удаляет его из очереди, чтобы избежать повторной обработки и освободить ресурсы.

HandleEvent() обрабатывает событие. Он анализирует тип события (what) и, в зависимости от него, вызывает соответствующие обработчики.

GetEvent() извлекает следующее событие из очереди событий и заполняет переданную структуру события. Возвращает true, если событие получено, и false, если событий нет.

Поле EndState используется в классах диалогов или окон для указания того, какой элемент завершил диалог. Содержит его обычно класс TDialog.

Функция Valid() проверяет корректность данных, введенных пользователем, прежде чем закрыть диалог или перейти к следующему шагу. Если проверка не пройдена, Valid() возвращает false и не позволяет закрыть окно.
