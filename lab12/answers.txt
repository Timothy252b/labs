Ассоциативный контейнер — это контейнер, который хранит элементы в отсортированном порядке и обеспечивает быстрый доступ к элементам по ключу, а не по позиции. В отличие от последовательных контейнеров, ассоциативные контейнеры автоматически упорядочивают элементы согласно заданному критерию.

Ассоциативные контейнеры библиотеки STL: set, multiset, map, multimap. Также сюда можно отнести unordered_set, unordered_multiset, unordered_map, unordered_multimap (неупорядоченные ассоциативные контейнеры, основанные на хешировании).

Доступ к элементам ассоциативного контейнера осуществляется по ключу с помощью оператора [] (для map), метода find(), методов lower_bound() и upper_bound(), а также с помощью итераторов, которые позволяют обходить элементы в отсортированном порядке.

Примеры методов: insert() — добавление элемента; erase() — удаление по ключу или итератору; find() — поиск по ключу; count() — подсчёт элементов с заданным ключом; lower_bound() — первый элемент не меньше ключа; upper_bound() — первый элемент больше ключа; equal_range() — диапазон элементов с данным ключом.

Создать контейнер map можно следующими способами:
map<int, string> m1; — пустой map с ключом int и значением string.
map<int, string> m2 = {{1, "one"}, {2, "two"}}; — инициализация списком.
map<int, string> m3(m2); — копирование.
map<int, string> m4(m2.begin(), m2.end()); — диапазоном итераторов.

По умолчанию элементы в map упорядочены по ключу в порядке возрастания с помощью компаратора less<Key> (от меньшего к большему). Чтобы изменить порядок на обратный, нужно использовать компаратор greater<Key>: map<int, string, greater<int>> m;

Операции, определённые для map: оператор [] для доступа и вставки; insert() и emplace() для добавления; erase() для удаления; find() для поиска; at() для доступа с проверкой границ; size(), empty(), clear(); итераторные операции begin(), end(), rbegin(), rend(); swap() для обмена содержимым; count() для проверки наличия ключа.

Функция для добавления элементов в map с помощью make_pair():
void addPair(map<int, string>& m, int key, const string& value) {
    m.insert(make_pair(key, value));
}

Функция для добавления элементов в map с помощью оператора []:
void addWithBracket(map<int, string>& m, int key, const string& value) {
    m[key] = value;
}

Функция для печати map с помощью итератора:
void printWithIterator(const map<int, string>& m) {
    for (auto it = m.begin(); it != m.end(); ++it) {
        cout << it->first << " -> " << it->second << endl;
    }
}
Или с использованием range-based for:
void printWithIterator(const map<int, string>& m) {
    for (const auto& p : m) {
        cout << p.first << " -> " << p.second << endl;
    }
}

Функция для печати map с помощью оператора [] (не рекомендуется, так как оператор [] в map для константного доступа не подходит, а для неконстантного создаёт элементы). Корректный вариант с использованием [] только для существующих ключей:
void printWithBracket(map<int, string>& m) {
    for (auto it = m.begin(); it != m.end(); ++it) {
        cout << it->first << " -> " << m[it->first] << endl;
    }
}

Отличия map и multimap: map хранит уникальные ключи (каждому ключу соответствует только одно значение), при вставке дублирующего ключа вставка игнорируется. multimap допускает хранение нескольких элементов с одинаковым ключом, предоставляет методы equal_range() для получения диапазона всех элементов с одним ключом, и не имеет оператора [].

Контейнер set — это ассоциативный контейнер, который хранит уникальные отсортированные ключи без ассоциированных с ними значений. Каждый элемент является одновременно и ключом, и значением.

Отличия map и set: map хранит пары ключ-значение, обеспечивая отображение из ключа в данные; set хранит только ключи (одиночные значения). К элементам map обращаются через first и second, к элементам set — непосредственно. map позволяет модифицировать значение, ассоциированное с ключом, но не сам ключ; set не позволяет модифицировать элементы, так как они сами являются ключами.

Создать контейнер set можно следующим образом:
set<int> s1; — пустой set целых чисел.
set<int> s2 = {1, 2, 3, 4}; — инициализация списком.
set<int> s3(s2); — копирование.
set<int> s4(s2.begin(), s2.end()); — диапазоном итераторов.
set<string, greater<string>> s5; — set строк в обратном порядке.

По умолчанию элементы в set упорядочены по возрастанию с помощью less<Key>. Чтобы изменить порядок на обратный, нужно указать компаратор greater<Key>: set<int, greater<int>> s;

Операции, определённые для set: insert() для добавления; erase() для удаления; find() для поиска; count() для проверки наличия; lower_bound(), upper_bound(), equal_range() для поиска диапазонов; size(), empty(), clear(); итераторные операции begin(), end(), rbegin(), rend(); swap().

Функция для добавления элементов в set:
void addToSet(set<int>& s, int value) {
    s.insert(value);
}
Для проверки успешности вставки:
bool addToSet(set<int>& s, int value) {
    return s.insert(value).second;
}
