Необходимо разработать иерархию классов с абстрактным базовым классом Person. Базовый класс должен содержать два закрытых поля: имя в виде строки и возраст в виде целого числа. Для доступа к этим полям требуются методы setName, setAge, getName, getAge. Метод Show должен быть объявлен как чисто виртуальный, что делает класс Person абстрактным. Это означает, что создать объект самого класса Person невозможно, но можно создавать объекты производных классов и работать с ними через указатели или ссылки на базовый класс.

Конструкторы класса Person должны быть определены: конструктор по умолчанию, инициализирующий имя значением Jane Doe и возрастом -1, конструктор с параметрами для установки имени и возраста, конструктор копирования. Деструктор должен быть виртуальным для корректного разрушения объектов производных классов.

Производный класс Student наследует от Person. Он добавляет два закрытых поля: предмет в виде строки и оценку в виде целого числа. Конструктор без параметров вызывает конструктор базового класса и инициализирует предмет значением Astrology, оценку значением -1. Конструктор с параметрами принимает имя, возраст, предмет и оценку, передает имя и возраст в конструктор базового класса и инициализирует собственные поля. Конструктор копирования вызывает конструктор копирования базового класса и копирует предмет и оценку.

Метод Show должен быть переопределен в классе Student. Он сначала вызывает метод Show базового класса, который выводит имя и возраст, а затем выводит предмет и оценку. Метод isGood анализирует значение оценки и выводит сообщение good student или bad student в зависимости от того, больше ли оценка трех или меньше.

Для класса Student также требуется перегрузить оператор вывода, который выводит все данные о студенте построчно.

Класс Vector предназначен для хранения массива указателей на объекты типа Person. Это позволяет хранить в одном контейнере объекты разных классов иерархии благодаря полиморфизму. Класс должен содержать три закрытых поля: указатель на указатель на Person, представляющий динамический массив, целое число size для хранения максимальной вместимости вектора, и целое число cur для хранения текущего количества элементов.

Конструктор по умолчанию инициализирует массив нулевым указателем, размер и текущее количество нулями. Конструктор с параметром a выделяет динамическую память под массив из a указателей на Person, устанавливает размер равным a и текущее количество равным нулю. Деструктор освобождает выделенную память с помощью оператора delete[].

Метод Add принимает указатель на Person и добавляет его в массив, если текущее количество элементов меньше максимального размера. В противном случае выводится сообщение об ошибке.

Оператор вывода для класса Vector должен быть перегружен как дружественная функция. Он проходит по всем элементам массива от нуля до cur и для каждого указателя вызывает метод Show. Благодаря виртуальности метода Show будет вызвана соответствующая версия для каждого объекта. Если в векторе хранятся объекты Student, то вызовется Show из класса Student.

Демонстрационная программа должна создать объекты Student. Создать вектор размером пять. Добавить в вектор указатель на объект Student несколько раз с помощью метода Add. Вывести содержимое вектора, что покажет работу перегруженного оператора вывода и продемонстрирует полиморфизм. Также программа должна показать работу метода Show при вызове через указатель на базовый класс и непосредственно для объектов Student, и работу метода isGood.
