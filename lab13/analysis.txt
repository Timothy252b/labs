Программа состоит из трех задач, демонстрирующих работу с различными контейнерами и адаптерами стандартной библиотеки C++ с использованием алгоритмов STL.

В первой задаче используется последовательный контейнер list для хранения объектов пользовательского типа Pair. Структура Pair содержит два целых поля first и second. Оператор меньше перегружен для сравнения только по полю first, что определяет порядок сортировки. Оператор равенства позволяет сравнивать пары поэлементно. Оператор вывода выводит пару в круглых скобках через запятую.

Исходный список содержит семь пар. Алгоритм replace_if заменяет все элементы, у которых поле first равно пяти, на пару с нулевыми значениями. Затем метод remove_if удаляет все элементы с нулевым полем first. Для сортировки используется метод sort списка с пользовательскими компараторами сначала по возрастанию, затем по убыванию поля first. Алгоритм find_if ищет первый элемент с полем first равным семи. Алгоритм count_if подсчитывает количество элементов с полем first равным двум. Для выполнения варианта задания используются алгоритмы minmax_element для нахождения минимального и максимального элементов по полю first и for_each для прибавления суммы минимального и максимального значений к полю first каждого элемента.

Во второй задаче используется адаптер контейнера priority_queue, настроенный как max-heap на основе поля first структуры Pair. В очередь помещаются пять пар. Для выполнения операций замены и удаления элементы извлекаются из очереди в вектор. Алгоритм replace_if заменяет все элементы с полем first равным единице на пару со значениями девять и девять. Алгоритм remove_if удаляет элементы с полем first равным девяти, после чего erase окончательно удаляет их из вектора. Сортировка выполняется алгоритмом sort с компараторами для возрастания и убывания по полю first. Алгоритм find_if ищет элемент с полем first равным двум. Алгоритм count_if подсчитывает элементы с полем first равным четырем. Для варианта задания находятся минимальный и максимальный элементы по полю first с помощью minmax_element, затем for_each прибавляет сумму этих значений к полю first каждого элемента.

В третьей задаче используется ассоциативный контейнер map, хранящий пары ключ-значение, где ключ и значение являются целыми числами. Исходный словарь содержит пять записей. Замена элемента выполняется созданием временного словаря, в который копируются все записи, кроме записи с ключом пять, вместо нее добавляется запись с ключом ноль. Удаление элементов в диапазоне ключей от двух до пяти включительно выполняется прямым обходом итератора с вызовом erase. Сортировка в map выполняется автоматически по возрастанию ключей, поэтому специальной сортировки не требуется. Поиск элемента по ключу выполняется методом find. Алгоритм count_if подсчитывает количество записей с положительным ключом. Для варианта задания вычисляется среднее арифметическое всех значений с помощью accumulate, затем в словарь добавляется новый элемент с ключом, равным целой части среднего, и значением ноль. Затем находятся минимальный и максимальный ключи через begin и rbegin, вычисляется их сумма, и создается новый словарь, в котором каждый ключ увеличен на эту сумму при сохранении соответствующих значений.

Основная функция последовательно вызывает все три задачи, каждая из которых выводит промежуточные и итоговые результаты своих операций. Программа демонстрирует применение стандартных алгоритмов STL для различных типов контейнеров.
