Что такое абстрактный тип данных Привести примеры АТД. Абстрактный тип данных это математическая модель, определяющая набор операций над данными, скрывающая детали реализации. Примеры стек с операциями push pop top, очередь с операциями enqueue dequeue, список с операциями insert delete find, множество с операциями add remove contains.

Привести примеры абстракции через параметризацию. Абстракция через параметризацию это создание шаблонных классов и функций. Примеры шаблонный класс List T где T может быть int double string, шаблонная функция sort T, класс Pair T1 T2, класс Array T.

Привести примеры абстракции через спецификацию. Абстракция через спецификацию это определение интерфейса класса через абстрактные классы или концепты. Примеры абстрактный класс Container с чисто виртуальными методами add remove get, интерфейс Iterable с методом iterator, абстрактный класс Stack с методами push pop.

Что такое контейнер Привести примеры. Контейнер это объект, предназначенный для хранения и управления коллекцией других объектов. Примеры массив int a 10, вектор vector int, список list int, множество set int, словарь map int string.

Какие группы операций выделяют в контейнерах? В контейнерах выделяют операции добавления элементов, удаления элементов, доступа к элементам, поиска элементов, получения размера контейнера, итерирования по элементам, сравнения контейнеров.

Какие виды доступа к элементам контейнера существуют Привести примеры. Существует прямой доступ по индексу в массивах и векторах, последовательный доступ через итераторы в списках и множествах, ассоциативный доступ по ключу в словарях и множествах, доступ через указатели.

Что такое итератор? Итератор это объект, предоставляющий унифицированный способ последовательного доступа к элементам контейнера без раскрытия его внутренней структуры, аналогичный указателю.

Каким образом может быть реализован итератор? Итератор может быть реализован как вложенный класс внутри контейнера, хранящий указатель на текущий элемент и перегружающий операции инкремента для перехода к следующему элементу, разыменования для доступа к значению, сравнения для проверки достижения конца.

Каким образом можно организовать объединение контейнеров? Объединение контейнеров можно организовать последовательным копированием элементов из одного контейнера в другой через итераторы и алгоритм copy, либо через метод insert, либо создавая новый контейнер и добавляя в него элементы из нескольких исходных.

Какой доступ к элементам предоставляет контейнер, состоящий из элементов ключ-значение? Контейнер ключ-значение предоставляет доступ по ключу, то есть по значению ключа можно получить связанное с ним значение за время O log n в сбалансированном дереве или O 1 в хеш-таблице.

Как называется контейнер, в котором вставка и удаление элементов выполняется на одном конце контейнера? Такой контейнер называется стек.

Какой из объектов a b c d является контейнером? Объект d int mas 100 является массивом, то есть контейнером. Объект a это отдельная переменная, объект b это неполное объявление, объект c это структура без массива.

Какой из объектов a b c d не является контейнером? Объект d int mas является отдельной переменной типа int, не контейнером. Объект a является массивом контейнером, объект b массив контейнер, объект c массив структур контейнер.

Контейнер реализован как динамический массив, в нем определена операция доступ по индексу. Каким будет доступ к элементам контейнера? Доступ будет прямым произвольным за постоянное время O 1.

Контейнер реализован как линейный список. Каким будет доступ к элементам контейнера? Доступ будет последовательным для доступа к элементу с заданным индексом необходимо пройти по списку от начала до нужной позиции за линейное время O n.
