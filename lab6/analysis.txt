Требуется разработать класс-контейнер List для хранения целочисленных значений с организацией двусвязного списка. Внутреннее представление списка должно быть реализовано через структуру Node, которая содержит целочисленное поле data, указатели на предыдущий и следующий узлы. Класс List должен хранить указатели на голову и хвост списка, а также целочисленное поле size для отслеживания количества элементов.

Для класса List необходимо реализовать конструктор по умолчанию, создающий пустой список. Конструктор с целочисленным параметром n должен создавать список из n элементов, каждый из которых инициализируется нулем. Конструктор копирования должен создавать глубокую копию переданного списка. Деструктор должен последовательно удалять все узлы списка, освобождая память.

Оператор присваивания должен выполнять проверку на самоприсваивание, затем очищать текущий список и копировать содержимое другого списка.

В классе должны быть реализованы базовые методы работы со списком. Метод pushBack добавляет новый элемент в конец списка. Метод pushFront добавляет элемент в начало. Метод popBack удаляет последний элемент, метод popFront удаляет первый элемент. Также требуются методы front и back для доступа к первому и последнему элементам, метод getSize для получения количества элементов, метод print для вывода содержимого списка.

Оператор доступа по индексу должен быть перегружен в двух версиях. Первая версия возвращает ссылку на элемент для изменения, вторая версия для константных объектов возвращает константную ссылку. Обе версии должны проверять, что индекс находится в допустимых пределах, и выбрасывать исключение при выходе за границы. Для доступа к элементу по индексу необходимо последовательно пройти от головы списка до нужной позиции.

Оператор приведения к целому типу должен возвращать текущий размер списка. Это позволяет использовать объект списка в контексте, где ожидается целое число, например, в арифметических выражениях или при выводе.

Внешняя функция operator* должна выполнять поэлементное умножение двух списков. Функция должна проверять, совпадают ли размеры списков. Если размеры различны, выводится сообщение об ошибке и возвращается пустой список. Если размеры совпадают, создается новый список того же размера, в котором каждый элемент i равен произведению i-го элемента первого списка на i-й элемент второго. Для доступа к элементам используется оператор индексации.

Для обеспечения последовательного доступа к элементам списка требуется реализовать класс-итератор ListIter. Итератор должен содержать указатель на узел списка. Дружественные отношения между классами List и ListIter позволяют итератору получать доступ к внутренней структуре списка. Конструкторы итератора должны позволять создавать итератор из указателя на узел, из ссылки на узел, а также из ссылки на список, при этом итератор настраивается на начало списка.

В классе итератора необходимо перегрузить оператор сложения с целым числом, который возвращает новый итератор, смещенный вправо на заданное количество позиций. Реализация должна последовательно проходить по указателям next от текущего узла. Если при переходе достигается конец списка, итератор должен указывать на нулевой указатель. Оператор разыменования должен возвращать ссылку на целочисленные данные узла, что позволяет читать или изменять значение элемента.

Демонстрационная программа должна создать два списка, заполнить их значениями, показать работу оператора умножения списков. Затем создать итератор для результирующего списка и с помощью сложения смещаться по элементам, выводя их значения на экран. Все операции должны быть проиллюстрированы на конкретных примерах.
