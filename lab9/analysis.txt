Требуется разработать класс List, реализующий контейнер для хранения целочисленных значений с динамическим выделением памяти. Класс должен содержать два закрытых поля: указатель на динамический массив целых чисел и целочисленное поле size для хранения текущего размера массива.

Конструктор по умолчанию принимает целочисленный параметр s, равный нулю, и выделяет память под массив указанного размера. При создании списка с отрицательным размером должна генерироваться исключительная ситуация. Конструктор копирования должен выполнять глубокое копирование данных из другого объекта. Деструктор освобождает выделенную память.

Оператор доступа по индексу должен быть перегружен в двух версиях для константных и неконстантных объектов. При обращении по индексу, выходящему за границы массива, должна генерироваться исключительная ситуация. Оператор приведения к целому типу должен возвращать текущий размер списка. Оператор умножения должен выполнять поэлементное умножение двух списков и возвращать новый список с результатами. Если списки имеют разный размер, должна генерироваться исключительная ситуация.

Оператор сдвига вправо на n позиций должен удалять из списка первые n элементов. Реализация заключается в создании нового временного списка размера size минус n, копировании в него элементов начиная с позиции n, затем присваивании содержимого временного списка текущему. При передаче отрицательного значения n или значения, превышающего размер списка, должна генерироваться исключительная ситуация. Оператор возвращает ссылку на текущий объект для поддержки цепочек операций.

Оператор присваивания должен выполнять проверку на самоприсваивание, освобождать старую память, копировать размер и данные из другого объекта.

Метод print выводит содержимое списка в квадратных скобках с элементами, разделенными запятыми. Метод fill заполняет все элементы списка заданным значением.

Для обработки исключительных ситуаций требуется определить иерархию пользовательских классов-исключений, унаследованных от стандартного класса exception. Класс ListIndexError предназначен для ситуаций выхода индекса за допустимые пределы. Класс ListSizeError для случаев создания списка с отрицательным размером. Класс ListMultiplicationError для ситуаций, когда списки имеют разный размер при попытке умножения. Каждый класс должен переопределять метод what, возвращающий соответствующее текстовое сообщение об ошибке.

В основной программе необходимо продемонстрировать создание списков, заполнение их значениями, вывод содержимого, получение размера с помощью оператора приведения, доступ к элементам по индексу, умножение списков, а также операцию сдвига вправо на заданное количество позиций.

Отдельно требуется протестировать обработку исключительных ситуаций в блоке try catch. Демонстрируются следующие случаи: выход индекса за пределы массива при обращении через оператор индексации, попытка создания списка с отрицательным размером, попытка умножения списков разной длины. Каждая ошибка перехватывается соответствующим типом исключения, и выводится информационное сообщение.

Дополнительно в программе демонстрируется работа с оператором присваивания при использовании операции сдвига, которая изменяет исходный список, создавая его уменьшенную копию. Все операции сопровождаются выводом на экран для визуального контроля корректности работы.
