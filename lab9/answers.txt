Исключение в C++ — это аномальная ситуация, возникающая во время выполнения программы (например, деление на ноль, выход за границы массива, ошибка выделения памяти), которая не может быть обработана в месте возникновения обычным способом. Исключение передаёт управление и информацию об ошибке в специально предназначенный для этого блок обработки.

Исключения позволяют разделить вычислительный процесс на две независимые части: основной алгоритм (логику программы) и обработку ошибочных ситуаций. Достоинства: код становится чище и легче для сопровождения, логика обработки ошибок не смешивается с бизнес-логикой; обработка ошибокцентрализована; можно передавать ошибку через несколько уровней вызовов без промежуточной проверки.

Оператор throw используется для генерации исключительной ситуации.

Контролируемый блок (try-блок) — это фрагмент кода, в котором могут возникнуть исключения. Он нужен, чтобы отделить потенциально проблемный код от кода обработки ошибок и позволить системе искать подходящий обработчик (catch) при генерации исключения.

Секция-ловушка (catch-блок) — это фрагмент кода, следующий за try, который перехватывает и обрабатывает исключения определённого типа. Она нужна для того, чтобы принять меры при возникновении конкретного типа исключения (вывести сообщение, освободить ресурсы, выполнить альтернативное действие).

Спецификация исключения в catch может иметь три формы: catch(...) — ловит любое исключение (используется, когда тип неизвестен или нужен гарантированный перехват всего); catch(тип) — ловит исключение конкретного типа (используется, когда известен тип ошибки и нужна разная логика); catch(тип& имя) — ловит по ссылке, что позволяет изменять перехваченный объект и избегает лишнего копирования (используется для работы с объектами классов).

Стандартный класс std::exception можно использовать для создания собственной иерархии исключений, отнаследовав от него свой класс.

Создать собственную иерархию исключений можно путем наследования от std::exception (или другого стандартного класса исключений) и переопределения виртуального метода what(), который возвращает пояснительную строку. При необходимости можно создать несколько уровней наследования для разных типов ошибок.

Если спецификация имеет вид void f1() throw(int, double);, то функция f1() может генерировать (пробрасывать) исключения только типа int или double. Любая попытка выбросить исключение другого типа приведет к вызову функции std::unexpected().

Если спецификация имеет вид void f1() throw();, то функция f1() не должна генерировать никаких исключений. Если исключение всё же будет выброшено, будет вызвана std::unexpected().

Исключение может генерироваться в любой части программы, где есть оператор throw, включая: тело функции, конструктор, деструктор, метод класса, блок try, блок catch (после частичной обработки исключение можно перебросить дальше), а также в инициализаторах глобальных и статических объектов.

Функция вычисления площади треугольника по формуле Герона в 4 вариантах:

Вариант 1. Без спецификации исключений:
double areaTriangle(double a, double b, double c) {
    if (a <= 0 || b <= 0 || c <= 0 || a + b <= c || a + c <= b || b + c <= a)
        throw "invalid sides";
    double p = (a + b + c) / 2;
    return sqrt(p * (p - a) * (p - b) * (p - c));
}

Вариант 2. Со спецификацией throw():
double areaTriangle(double a, double b, double c) throw() {
    if (a <= 0 || b <= 0 || c <= 0 || a + b <= c || a + c <= b || b + c <= a)
        return -1; // вместо исключения возвращаем ошибку
    double p = (a + b + c) / 2;
    return sqrt(p * (p - a) * (p - b) * (p - c));
}

Вариант 3. С конкретной спецификацией со стандартным исключением std::invalid_argument:
#include <stdexcept>
double areaTriangle(double a, double b, double c) throw(std::invalid_argument) {
    if (a <= 0 || b <= 0 || c <= 0 || a + b <= c || a + c <= b || b + c <= a)
        throw std::invalid_argument("Sides do not form a valid triangle");
    double p = (a + b + c) / 2;
    return sqrt(p * (p - a) * (p - b) * (p - c));
}

Вариант 4. Спецификация с собственным реализованным исключением:
#include <stdexcept>
class TriangleError : public std::exception {
    const char* what() const noexcept override {
        return "Triangle inequality violation or non-positive side";
    }
};
double areaTriangle(double a, double b, double c) throw(TriangleError) {
    if (a <= 0 || b <= 0 || c <= 0 || a + b <= c || a + c <= b || b + c <= a)
        throw TriangleError();
    double p = (a + b + c) / 2;
    return sqrt(p * (p - a) * (p - b) * (p - c));
}
